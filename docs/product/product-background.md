# 产品背景

## 目标用户

主要是作者本人（Yannis），将博客作为个人知识库使用。次要受众为同样在学习 C 语言、Java、Python、高等数学等基础课程的在校学生或自学者。

### 用户画像

- **主要用户**：Yannis 本人，计算机相关专业学生，需要整理和回顾学习笔记
- **次要用户**：搜索引擎导流的技术初学者，寻找特定知识点的参考

## 用户痛点

1. **笔记分散**：学习笔记散落在各种平台和本地文件中，难以统一检索和回顾
2. **缺乏结构化**：零散的笔记缺少分类和标签体系，知识点之间缺乏关联
3. **不便分享**：本地笔记无法方便地分享给同学或其他学习者

## 行业现状与核心矛盾

个人技术博客领域工具成熟（WordPress、Hexo、Hugo、Jekyll 等），但核心矛盾在于：**搭建博客容易，持续产出高质量内容难**。大量技术博客在初期搭建后逐渐荒废，真正有价值的是内容本身而非技术框架。

本博客选择 Hexo + GitHub Pages 的轻量方案，将精力集中在内容产出而非平台维护上。

## 解决方案

基于以上痛点，本博客通过以下方式解决问题：

- **集中存储**：所有笔记统一发布到博客站点，随时可访问
- **结构化组织**：通过分类、标签、归档体系组织内容
- **公开分享**：部署在 GitHub Pages 上，任何人可访问
- **技术友好**：支持公式渲染、代码高亮等技术写作需求
