# 产品宪法

> 基于 Hexo + Chic 主题的个人技术博客，记录学习笔记和成长历程。

## 1. 产品定义

Yannis's Blog 是一个个人技术博客静态站点，用于集中记录和分享技术学习笔记（C 语言、Java、Python、高等数学等），同时承载个人成长历程。站点基于 Hexo 静态博客生成器和 Chic 主题构建，部署在 GitHub Pages 上。

## 2. 核心能力

- **技术笔记记录与展示**：发布和展示技术学习笔记，涵盖编程语言、算法、数学等领域
- **文章分类与归档**：按分类、标签、时间归档浏览文章，方便检索
- **技术内容渲染**：支持数学公式（MathJax）、代码高亮、文章目录（tocbot）等技术写作需求
- **读者互动与统计**：通过 Gitalk 评论系统实现读者互动，通过不蒜子统计访客数据

## 3. 核心问题

学习笔记分散在各处，缺乏统一的整理和展示平台。本博客将零散的学习记录集中到一个结构化的站点中，既方便自己回顾，也能分享给其他学习者。

## 4. 产品边界 (What NOT)

我们明确**不做**以下内容：

- **不做多用户平台**：这是个人博客，不做多用户内容发布平台
- **不做商业化变现**：不做电商、广告、付费内容等商业化功能
- **不做动态后端服务**：保持纯静态站点，不引入复杂的后端服务
- **不做非技术内容**：聚焦技术学习领域，不做与技术无关的生活日志

## 5. 决策原则

当需求冲突时，遵循以下优先级：

1. **内容质量 > 更新频率**：优先保证笔记内容的准确性和深度，而非追求发布数量
2. **简洁 > 复杂**：优先保持简洁的设计和结构，而非添加复杂功能
3. **阅读体验 > 视觉效果**：优先保证文章的可读性和排版，而非视觉炫技
4. **稳定性 > 新功能**：优先保证站点稳定运行和加载速度，而非引入新功能

## 6. 需求分类模型

基于以上原则，将需求分为四类：

### A 类：核心产品能力
- 直接服务于技术笔记的记录、展示和检索
- 提升文章阅读体验（排版、公式、代码高亮）
- 优先级最高

### B 类：阶段性实验
- 符合博客方向但价值不确定的功能（如站内搜索优化、RSS 订阅）
- 需要小规模验证
- 优先级中

### C 类：场景定制需求
- 仅对特定场景有价值（如特定主题的专题页）
- 不具备通用性
- 优先级低

### D 类：噪音/过早需求
- 与个人技术博客定位无关（如电商功能、用户注册）
- 触碰产品边界
- 拒绝
